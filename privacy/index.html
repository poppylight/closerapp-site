6. How Your Data Is Stored

6.1 Local Storage (Default)

All financial data you enter is stored exclusively on your device using Apple's secure storage frameworks (UserDefaults with App Group containers, and Keychain Services for sensitive data such as API keys). We have no access to this data — it never leaves your device unless you explicitly enable iCloud Sync.

6.2 Optional iCloud Sync (Pro Feature)

If you enable iCloud Sync, your data is transmitted to and stored in your personal Apple iCloud account via Apple's CloudKit framework. Key points:

  • Data is encrypted in transit (TLS 1.2+) and at rest (AES-256)
  • Data resides in your iCloud account — we have no server-side access
  • iCloud Sync is governed by Apple's Privacy Policy
  • You can disable sync or delete synced data at any time from iOS Settings
  • No personal identifiers (names, emails) are stored alongside your financial data

6.3 Keychain & Secure Storage

Sensitive application data (e.g., API key obfuscation seeds) is stored using Apple's Keychain Services, which provides hardware-backed encryption via the device's Secure Enclave where available.

14. Security Measures

We implement industry-standard technical and organisational measures to protect your data:

  • Encryption at rest: Sensitive data stored using Apple's Keychain Services with hardware-backed encryption
  • Encryption in transit: All network requests use HTTPS (TLS 1.2 or later)
  • API key protection: API keys are obfuscated using XOR encryption and stored in the Keychain, never in plain text
  • Biometric protection: Optional Face ID / Touch ID lock (processing handled by Apple's Secure Enclave — biometric data never leaves your device)
  • Privacy screen: App content is automatically hidden in the iOS app switcher to prevent shoulder-surfing
  • No remote servers: We do not operate servers that store your data, eliminating server-side breach risk
  • Debug safeguards: All diagnostic logging is disabled in production builds
